# 05 — Como repetir este processo em outros sites

Este documento explica como usar este mesmo método
para criar outros sites institucionais de forma rápida e organizada.

A ideia é repetir o processo,
não reinventar tudo do zero.

---

## Quando usar este método?

Este método é ideal para:
- sites institucionais simples
- projetos de associações
- cooperativas
- pequenos negócios
- parceiros do Observatório

Ele não foi feito para sistemas complexos,
mas para sites claros e objetivos.

---

## Estrutura mínima para novos sites

Para outros sites, a estrutura pode ser ainda mais simples:

site/
├── index.html
├── textos/
│ ├── home.md
│ ├── sobre.md
│ └── contato.md
└── docs/

Nem todos os sites precisam de:
- parceiros
- galeria
- livros
- documentação extensa

---

## Passo a passo para repetir

1. Copiar a estrutura base do projeto
2. Criar um novo repositório no GitHub
3. Ajustar o nome do site no `index.html`
4. Trocar os textos dentro da pasta `textos/`
5. Conectar o novo repositório à Vercel
6. Publicar o site

Seguindo esses passos, o site fica no ar rapidamente.

---

## Tempo médio de criação

Depois que o método é entendido:

- Criação do site: 20 a 30 minutos
- Ajustes de texto: conforme necessidade
- Visual: pode ser feito depois

O mais importante é:
👉 colocar no ar primeiro

---

## O que não precisa ser repetido?

Para os sites filhos:
- não é necessário toda a documentação
- não é necessário estrutura complexa
- não é necessário pensar tudo de novo

A base já está pronta.

---

## Resumo final

Este método permite:
- criar sites com calma
- evitar bagunça
- ensinar outras pessoas
- crescer sem ansiedade

Tudo começa com organização.
